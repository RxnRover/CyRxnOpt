.. _code_conventions:

Code Conventions
================

.. todo::

    We conform to the `PEP 8 <https://peps.python.org/pep-0008/>`_ style
    guide for Python, specifically using `current Black code style
    <https://black.readthedocs.io/en/stable/the_black_code_style/current_style.html>`_.
    The only exception is the 80 char line width.

.. todo::

    Code formatter: `Black <https://black.readthedocs.io/en/stable/>`_
    Import sorting and formatting: `isort <https://pycqa.github.io/isort/>`_
    Code checking: `flake8 <https://github.com/pycqa/flake8>`_
    These are all called automatically from pre-commit.

.. todo::

    For files, follow PEP 8 naming conventions, don't write monolithic
    files, classes, or functions.
